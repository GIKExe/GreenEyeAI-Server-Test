WebSocket подключение

ws://<server_ip>:8000/ws/esp?device_id=esp_01

Форматы сообщений (все через один WebSocket)

1. ESP -> Сервер (показания датчиков)

{
	"type": "sensors",
	"temperature": 23.5,
	"humidity": 65
}

2. ESP -> Сервер (подтверждение команды)

{
	"type": "command_ack",
	"command_id": 123,
	"status": "executed"
}
3. ESP -> Сервер (запрос команд)

{
	"type": "get_commands"
}

4. Сервер -> ESP (команда)

{
	"type": "command",
	"command_id": 123,
	"device": "light",
	"action": "on"
}

5. Сервер -> ESP (ответ на запрос команд)

{
	"type": "commands_list",
	"commands": [
		{"id": 1, "type": "light_on", "params": {}},
		{"id": 2, "type": "fan_off", "params": {}},
		{"id": 3, "type": "water_on", "params": {"duration": 30}}
	]
}


Полная схема общения

ESP → Сервер: {"type": "get_commands"}
Сервер → ESP: {"type": "commands_list", "commands": [...]}

ESP → Сервер: {"type": "sensors", "temperature": 23.5, "humidity": 65}

ESP → Сервер: {"type": "command_ack", "command_id": 123, "status": "executed"}
